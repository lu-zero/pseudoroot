# Benchmark results: 2026-07-02 unslop structural pass

Captured after a structural cleanup pass (dead-code removal, DRY-up of
duplicated syscall wrappers, framing/RPC boilerplate consolidation — commit
`1cfd1da`, atop `a762be5`) to confirm the restructuring had no performance
impact. No behaviour changes were intended; `cargo test
--workspace` and `cargo clippy --all-targets --workspace -- -D warnings` both
stayed green throughout.

Notable structural changes exercised by these runs:

- `pseudoroot-core/src/shm_map.rs`: `get_inode`/`upsert_chown`/`upsert_inode`/
  `remove_inode` now share one `probe()` linear-probe walk instead of each
  reimplementing it — this is the hot path for every `stat()` and `chown()`
  under SHM session mode (the default on Linux), so it's the change most
  worth re-benchmarking.
- `pseudoroot-lib/src/ownership.rs`: the `record_chown_{at,path,fd}`,
  `record_chmod_{at,path,fd}`, and `fake_{set,get,list,remove}xattr_{path,fd}`
  wrapper families were collapsed onto a shared `StatSource` enum instead of
  each hand-rolling its own path/fd/at stat resolution.
- `pseudoroot-core/src/daemon_client.rs` and `daemon_server.rs`: RPC
  boilerplate and socket-framing duplication removed (`daemon_request`
  helper, shared `write_framed`/`read_framed`, shared `bind_listener`).

None of this should be observable in throughput — it's the same operations
through fewer, shared code paths — and the numbers below confirm that.

## System

Same physical host as the [2026-07-01](2026-07-01.md) run:

- Host: Gentoo Linux, kernel `7.0.1-gentoo` (aarch64)
- CPU: Ampere-1a, 128 cores, 1 socket, no SMT (`On-line CPU(s): 0-127`)
- RAM: 255 GiB
- rustc/cargo: 1.95.0 (`59807616e` / `f2d3ce0bd`)
- pseudoroot commit: `1cfd1da` (release build, `cargo build --release --workspace`;
  benchmarks were run against this commit's working tree just before it was committed)
- fakeroost commit: `01d6a65` (sibling checkout at `../fakeroost`, release build)
- `fakeroot` (classic C tool): system-installed, version 1.33

All three runs used the default session mode (SHM-backed inode map on Linux,
`PSEUDOROOT_SESSION_SHM` unset) — the mode that exercises the refactored
`shm_map.rs` probe path.

## Takeaway

The unslop structural pass — probe-loop dedup in `shm_map.rs`, `StatSource`
consolidation in `ownership.rs`, RPC/framing dedup in `daemon_client.rs` /
`daemon_server.rs`, dead-code removal (`FileOwnership`/`UidGidMap`), and a
clap `RunArgs` flatten in the CLI — is behaviour- and performance-neutral.
stat() throughput, install/tar correctness, and compile-workload overhead all
match the pre-refactor baseline within normal run-to-run variance.
